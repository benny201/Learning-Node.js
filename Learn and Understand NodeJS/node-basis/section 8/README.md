# NPM

## Semantic versioning
Major.Minor.Patch
e.g. 1.7.2
* Major: big change(code maybe break) 1.7.2 -> 2.0.0
* Minor: add new features(still backward compatible) -> 1.7.2 -> 1.8.0
* Patch: some bugs fixed -> 1.7.2 -> 1.7.3

### auto update of package
^ 1.7.2 : the carrot ^.
